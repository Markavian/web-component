Handlebar Errors

Handlebars throws JavaScript errors that are uncaught, and cause the page to crash. Broken Template

Empty tags Terminate Execution

Nothing renders after the first self closing tag.

Resolution

Can't fix, self-closing tags are not advised, nor supported by most major browsers: * http://www.w3.org/TR/html4/appendix/notes.html#h-B.3.3 * http://stackoverflow.com/questions/3558119/are-self-closing-tags-valid-in-html5 * http://dev.w3.org/html5/spec-author-view/syntax.html#syntax-start-tag They're known as void elements, and include: * area, base, br, col, embed, hr, img, input, keygen, link, menuitem, meta, param, source, track, wbr * http://www.w3.org/html/wg/drafts/html/master/single-page.html#void-elements > Void elements can't have any contents (since there's no end tag, no content can be put between the start tag and the end tag). Web Component JS doesn't perform any direct interactions with these elements, instead, the DOM of the browser strips them out before any JavaScript can kick in.
